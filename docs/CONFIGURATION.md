# Configuration Reference

Полный справочник по всем параметрам конфигурации `ai`.

---

## Обзор

`ai` не использует конфигурационные форматы вроде TOML, YAML или INI.
Конфигурация состоит из двух слоёв:

1. **Переменные окружения** — приоритет выше, действуют на текущую сессию
2. **Файлы в `~/.config/ai/`** — персистентное хранилище

Жёстко заданные внутренние константы (`MODEL`, `MAX_TOK` и лимиты инструментов)
не вынесены в конфиг намеренно — они меняются только правкой кода (см. ADR-001 в DECISIONS.md).

---

## Файловая структура

```
~/.config/ai/
├── api_key        # Обязательно. Ключ Anthropic API. chmod 600.
└── system_prompt  # Необязательно. Полная замена системного промпта.
```

Каталог `~/.config/ai/` создаётся автоматически при первом запуске `ai` или при установке через `install.sh`.

---

## Переменные окружения

### `ANTHROPIC_API_KEY`

| Параметр | Значение |
|---|---|
| Тип | строка |
| Формат | `sk-ant-...` |
| Приоритет | выше файла `api_key` |
| Где читается | `read_api_key()` в `ai.py` |

Если задана — файл `~/.config/ai/api_key` игнорируется.

```bash
# Разовый запуск с ключом из окружения
ANTHROPIC_API_KEY="sk-ant-..." ai "сколько места на диске?"

# Постоянно для текущей сессии
export ANTHROPIC_API_KEY="sk-ant-..."
ai "привет"

# Добавить в shell-конфиг (действует во всех сессиях)
echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.bashrc
```

Используйте переменную окружения когда:
- Ключ управляется системой секретов (vault, systemd credentials)
- Нужен разный ключ на разных серверах без редактирования файлов
- Запускаете `ai` из скриптов или CI

### `AI_INSTALL_DIR`

| Параметр | Значение |
|---|---|
| Тип | путь |
| По умолчанию | `$HOME/.local/bin` |
| Где читается | `install.sh` (только при установке) |

Переопределяет каталог куда `install.sh` кладёт исполняемый файл `ai`.

```bash
# Установить в /usr/local/bin (нужен sudo для записи)
AI_INSTALL_DIR="/usr/local/bin" \
  curl -fsSL https://raw.githubusercontent.com/mutiev/ai-command-helper/main/install.sh | bash
```

Не влияет на работу уже установленного `ai` — только на `install.sh`.

## Файлы конфигурации

### `~/.config/ai/api_key`

Ключ Anthropic API для аутентификации запросов.

**Формат:** одна строка, только ключ, без пробелов и переносов.

```
sk-ant-api03-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**Права доступа:** `600` (только владелец может читать и писать).
Файл создаётся с правами `600` автоматически при:
- Установке через `install.sh` с указанием ключа
- Первом запуске `ai` при интерактивном вводе через `getpass`

**Приоритет:** ниже переменной окружения `ANTHROPIC_API_KEY`.

**Способы создать/обновить:**

```bash
# Вручную
echo 'sk-ant-...' > ~/.config/ai/api_key
chmod 600 ~/.config/ai/api_key

# Через встроенный мастер настройки
ai --setup

# При установке (интерактивный режим install.sh)
bash /tmp/install.sh

# При установке через переменную окружения
ANTHROPIC_API_KEY="sk-ant-..." \
  curl -fsSL .../install.sh | bash
```

**Проверить текущий ключ:**
```bash
cat ~/.config/ai/api_key          # показать
wc -c ~/.config/ai/api_key        # длина (не показывая ключ)
ls -la ~/.config/ai/api_key       # проверить права
```

### `~/.config/ai/system_prompt`

Кастомный системный промпт — **полностью заменяет** `DEFAULT_SYSTEM`.

**Формат:** произвольный текст, UTF-8.

**Если файл отсутствует** — используется `DEFAULT_SYSTEM` из кода:

```
Ты — ассистент в терминале Linux. Пользователь работает по SSH.
Правила:
1. Когда задача решается командой — выдавай ТОЛЬКО команду (или несколько),
   без лишних объяснений, если не просят объяснений.
2. Обворачивай команды в ```bash``` блоки.
3. Если нужно посмотреть другие файлы/каталоги — используй инструменты.
4. Будь лаконичен. Терминал — не место для эссе.
5. Если задача неоднозначна — сначала уточни, потом предлагай решение.
```

**Важно:** файл заменяет промпт **целиком**. Контекст файловой системы
(листинг cwd, hostname, uname) добавляется поверх в любом случае — его убрать нельзя.

**Примеры использования:**

```bash
# DevOps-профиль: только команды, без лирики
cat > ~/.config/ai/system_prompt << 'EOF'
Ты — DevOps-инженер. Отвечай только командами в ```bash``` блоках.
Никаких объяснений если не просят. Никаких вступлений.
Система: Ubuntu 22.04 LTS, systemd, nginx, docker.
EOF
```

```bash
# Профиль для разработчика Python
cat > ~/.config/ai/system_prompt << 'EOF'
Ты — Python-разработчик. Отвечаешь на русском.
При работе с кодом — предпочитай стандартную библиотеку.
Всегда предлагай обработку ошибок. Используй f-строки, а не format().
EOF
```

```bash
# Английский язык для команды
cat > ~/.config/ai/system_prompt << 'EOF'
You are a Linux terminal assistant. Respond in English only.
Provide commands in ```bash``` blocks. Be concise.
EOF
```

**Удалить** (вернуться к дефолту):
```bash
rm ~/.config/ai/system_prompt
```

**Известное ограничение:** нет механизма "добавить к дефолту" — только полная замена.
Если нужно дополнить дефолтный промпт одним правилом — скопируйте `DEFAULT_SYSTEM` из `ai.py`
в файл и допишите своё правило в конец (см. ADR-009 в DECISIONS.md).

## Внутренние константы

Жёстко заданы в коде `ai.py`. Не вынесены в конфиг намеренно — изменение требует осознанного редактирования скрипта.

| Константа | Значение | Описание |
|---|---|---|
| `MODEL` | `claude-sonnet-4-5` | Модель Anthropic API |
| `MAX_TOK` | `4096` | Максимум токенов в ответе |
| `CONFIG_DIR` | `~/.config/ai` | Каталог конфигурации |
| `KEY_FILE` | `~/.config/ai/api_key` | Путь к файлу ключа |
| `SYSTEM_FILE` | `~/.config/ai/system_prompt` | Путь к файлу промпта |

Константы инструментов (см. подробнее в `docs/TOOLS.md`):

| Константа | Значение | Описание |
|---|---|---|
| `list_dir.max_entries` | `200` | Лимит записей в листинге каталога |
| `read_file_safe.max_bytes` | `32 768` | Лимит чтения файла (32 КБ) |
| `run_shell.timeout` | `10` | Таймаут выполнения команды (секунды) |

**Как изменить константу:**

```bash
# Например, сменить модель на Opus
sed -i 's/MODEL   = "claude-sonnet-4-5"/MODEL   = "claude-opus-4-5"/' ~/.local/bin/ai
```

## Порядок приоритетов (API-ключ)

```
ANTHROPIC_API_KEY (env)          ← наивысший приоритет
        ↓ не задана
~/.config/ai/api_key             ← персистентный файл
        ↓ не существует или пуст
getpass() → сохранить в файл     ← интерактивный запрос при первом запуске
        ↓ stdin не TTY
вывод инструкции + exit(1)       ← нет способа получить ключ
```

## Полная карта всего что можно настроить

```
┌─────────────────────────────────────────────────────┐
│              ЧТО НАСТРАИВАЕТСЯ                       │
├────────────────────────┬────────────────────────────┤
│ Поведение модели       │ ~/.config/ai/system_prompt  │
│ API-ключ               │ ~/.config/ai/api_key        │
│                        │ $ANTHROPIC_API_KEY          │
│ Каталог установки      │ $AI_INSTALL_DIR (install.sh)│
├────────────────────────┼────────────────────────────┤
│              ЧТО НЕ НАСТРАИВАЕТСЯ                    │
├────────────────────────┬────────────────────────────┤
│ Модель                 │ хардкод MODEL в ai.py       │
│ Лимит токенов          │ хардкод MAX_TOK в ai.py     │
│ Лимиты инструментов    │ хардкод в функциях ai.py    │
│ Каталог конфига        │ хардкод ~/.config/ai        │
│ Контекст ФС в промпте  │ всегда добавляется, убрать  │
│                        │ нельзя без правки кода      │
└────────────────────────┴────────────────────────────┘
```
