# ai — Claude в терминале

Лёгкий CLI-ассистент на базе [Claude](https://anthropic.com) для работы по SSH.  
Один Python-файл, никаких демонов, никакого сервера — просто команда `ai`.

```
$ ai "почему диск заполнен?"

╭─ bash
du -sh /* 2>/dev/null | sort -rh | head -20
╰─────
```

---

## Возможности

- **Однострочный вопрос** — задаёте вопрос, получаете ответ или готовую команду
- **Интерактивный режим** — диалог с памятью сессии, как чат
- **Контекст файловой системы** — модель автоматически видит файлы в рабочем каталоге
- **Tool use** — модель сама запрашивает содержимое файлов и каталогов при необходимости
- **Системная информация** — hostname и uname передаются в каждый запрос
- **Подсветка команд** — блоки `bash` выделяются цветом в терминале
- **Кастомный системный промпт** — можно заменить поведение модели под себя

## Установка

### Через curl (рекомендуется)

```bash
curl -fsSL https://raw.githubusercontent.com/mutiev/ai-command-helper/main/install.sh | bash
```

Скрипт:
- Проверит наличие Python 3.8+
- Скачает `ai` в `~/.local/bin/`
- Установит зависимость `anthropic`
- Добавит `~/.local/bin` в `PATH` если нужно
- Спросит API-ключ (если запущен интерактивно)

### С ключом в одну строку

```bash
ANTHROPIC_API_KEY="sk-ant-..." \
  curl -fsSL https://raw.githubusercontent.com/YOUR_USER/ai/main/install.sh | bash
```

### Вручную

```bash
# Скачать скрипт
curl -fsSL https://raw.githubusercontent.com/YOUR_USER/ai/main/ai \
  -o ~/.local/bin/ai && chmod +x ~/.local/bin/ai

# Сохранить ключ
mkdir -p ~/.config/ai
echo 'sk-ant-ваш-ключ' > ~/.config/ai/api_key
chmod 600 ~/.config/ai/api_key
```

### Обновление

Повторно запустите install.sh — скрипт заменит предыдущую версию.

```bash
curl -fsSL https://raw.githubusercontent.com/YOUR_USER/ai/main/install.sh | bash
```

## API-ключ

Получить ключ: <https://console.anthropic.com/settings/keys>

Ключ можно передать двумя способами:

```bash
# 1. Файл (рекомендуется)
echo 'sk-ant-...' > ~/.config/ai/api_key
chmod 600 ~/.config/ai/api_key

# 2. Переменная окружения
export ANTHROPIC_API_KEY="sk-ant-..."
```

Переменная окружения имеет приоритет над файлом.

## Использование

```
ai [ВОПРОС] [-d КАТАЛОГ] [-f ФАЙЛ] [--setup]
```

### Один вопрос

```bash
ai "как найти процессы, которые больше всего едят RAM?"
ai "объясни exit code 137"
ai "напиши regex для IPv4 адреса"
```

### С дополнительным контекстом

```bash
# Показать модели содержимое другого каталога
ai -d /etc/nginx "почему nginx не стартует?"

# Показать конкретный файл
ai -f /var/log/syslog "что за ошибки в последний час?"

# Комбинировать флаги
ai -d /opt/myapp -f /opt/myapp/config.yml "как изменить порт?"
```

### Интерактивный режим

```bash
ai
```

Запускает диалог с памятью — модель помнит контекст разговора в рамках сессии.

```
Claude AI · терминал-ассистент
  Каталог: /home/user/project  |  Модель: claude-sonnet-4-5
  Введите вопрос или 'exit' для выхода

❯ что за файлы тут лежат?
❯ какой из них самый большой?
❯ покажи первые 50 строк
❯ exit
```

### Первичная настройка

```bash
ai --setup
```

## Как модель использует инструменты

Модель имеет доступ к трём инструментам, которые вызывает автоматически при необходимости:

| Инструмент | Что делает |
|---|---|
| `list_directory` | Показывает содержимое каталога |
| `read_file` | Читает текстовый файл (до 32 КБ) |
| `run_command` | Выполняет read-only команды (`df`, `ps`, `uname`, `systemctl status` и т.п.) |

Пример — модель сама находит нужные файлы:

```
$ cd /opt/myapp && ai "почему сервис крашится?"

  → list_directory({"path": "/opt/myapp"})
  → read_file({"path": "/opt/myapp/logs/error.log"})
  ⚙ systemctl status myapp

В логах видна ошибка ENOMEM — процессу не хватает памяти.

╭─ bash
journalctl -u myapp --since "1 hour ago" | grep -i "oom\|memory\|killed"
╰─────
```

## Конфигурация

Все файлы конфигурации находятся в `~/.config/ai/`:

| Файл | Назначение |
|---|---|
| `api_key` | Ключ Anthropic API (режим 600) |
| `system_prompt` | Кастомный системный промпт (необязательно) |

### Кастомный системный промпт

```bash
cat > ~/.config/ai/system_prompt << 'EOF'
Ты — DevOps-ассистент. Отвечай только на русском.
При ответе всегда учитывай, что сервер работает под Ubuntu 22.04.
Команды давай сразу, без вступлений.
EOF
```

## Требования

- Python 3.8+
- `curl` или `wget` (для установки)
- `pip` (установится автоматически через `ensurepip` если нет)
- Ключ Anthropic API ([получить](https://console.anthropic.com/settings/keys))

Зависимость `anthropic` устанавливается автоматически при первом запуске, если её нет.
